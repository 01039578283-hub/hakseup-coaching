# 새 홈페이지2 대량 페이지 생성 규칙

이 문서는 `새 홈페이지2`에서 371개 동네 페이지와 자식페이지를 생성할 때 반드시 따를 규칙이다.

## 1. 기본 페이지 구조

- 생성 위치는 기본적으로 `전국학원/` 하위로 둔다.
- 동네 페이지와 자식페이지 모두 기존 `새 홈페이지2` 디자인 톤을 따른다.
- 상단 메뉴, 우측 하단 버튼, 상담 CTA, footer 전화번호는 기존 페이지와 동일하게 유지한다.
- CSV로 생성할 경우:
  - A열: 페이지 제목
  - B열: 숨김 또는 대표 이미지 링크
  - C/D열: 본문 이미지
  - E열: 본문 원고 HTML
  - F열: 센터 위치 안내 HTML
  - G열: 부모 슬러그/경로
  - H열: 현재 페이지 슬러그

## 2. Title / H1 / 본문 제목

- `<title>`은 `{페이지 타이틀} | {홈페이지 제목}` 형식으로 만든다.
- H1은 페이지 타이틀을 기준으로 하되, 무리하게 긴 경우 자연스럽게 줄바꿈한다.
- H2/H3는 한 검색어를 흐리지 않도록 페이지 타이틀과 주제를 중심으로 작성한다.
- 기존 홈페이지 문구를 그대로 복사하지 않고, `새 홈페이지2`의 말투에 맞춰 상담형·관리형 문장으로 각색한다.

## 3. local / seoul 이미지 alt 규칙

앞으로 `assets/centers/common/local.jpg`, `assets/centers/common/seoul.jpg` 또는 CSV에서 `local`, `seoul` 본문 이미지가 사용되면 alt는 아래 형식으로 처리한다.

```text
{페이지 타이틀} {홈페이지 제목}
```

예시:

```html
<img src="../../assets/centers/common/local.jpg" alt="구파발 중2 영어학원 학습코칭 연구소">
<img src="../../assets/centers/common/seoul.jpg" alt="명일동 고1 수학학원 학습코칭 연구소">
```

- 여기서 `홈페이지 제목`은 현재 기준 `학습코칭 연구소`다.
- 나중에 사이트명이 바뀌면 alt도 `{페이지 타이틀} {새 홈페이지 제목}`으로 일괄 변경한다.
- 지도 이미지는 별도 규칙으로 `{페이지 타이틀} 위치 안내`를 사용한다.

## 4. FAQ 생성 규칙

- 각 동네 페이지와 자식페이지에는 화면에 보이는 FAQ 섹션을 넣는다.
- FAQ JSON-LD도 반드시 함께 넣는다.
- 기존 홈페이지에서 사용했던 질문/답변을 그대로 복사하지 않고, 새 홈페이지2의 문장 톤에 맞춰 각색한다.
- 페이지별로 4~5개를 선택한다.
- 선택 방식은 페이지 경로를 seed로 사용해 deterministic하게 고른다.
  - 같은 페이지는 다시 생성해도 같은 FAQ가 나오게 한다.
  - 서로 다른 페이지는 조합이 조금씩 달라지게 한다.
- FAQ 화면 내용과 JSON-LD `FAQPage` 내용은 반드시 일치해야 한다.

## 5. 후기/평점 생성 규칙

- 후기 섹션은 실제 화면에 보이게 넣고, JSON-LD에도 같은 후기를 넣는다.
- 기존 홈페이지처럼 6개 후기를 기본 단위로 사용한다.
- 별점은 화면과 JSON-LD를 반드시 일치시킨다.
- 기본 규칙:
  - 후기 6개 중 5개는 5점
  - 후기 6개 중 1개는 4점
  - 평균 평점은 4.8
  - `ratingCount`와 `reviewCount`는 6
- 4점 후기의 위치는 페이지 경로를 seed로 정해 페이지마다 자연스럽게 달라지게 한다.
- 후기는 기존 홈페이지 문구를 그대로 복사하지 않고, 플래너·오답·상담·학습 습관 중심으로 각색한다.

## 6. JSON-LD 기본 구조

각 생성 페이지는 가능한 한 하나의 `application/ld+json` 또는 역할별 script로 아래 구조를 포함한다.

- `EducationalOrganization`
  - `name`
  - `url`
  - `telephone`
  - `address`가 있으면 `PostalAddress`
  - `openingHours`: `Mo-Sa 12:00-24:00`
  - `areaServed`
  - `aggregateRating`
  - `review`
  - 교육지원청 등록번호가 화면에 있으면 `identifier`
  - 교습비 표가 화면에 있으면 `hasOfferCatalog`
- `WebPage`
- `BreadcrumbList`
- `Article`
  - E열 원고가 화면에 들어간 경우 화면 원고 요약과 대표 이미지 기준으로 생성
- `FAQPage`
  - 화면 FAQ와 동일해야 함

## 7. SEO / 화면 일치 검증

생성 후 반드시 확인한다.

- `title`, H1, 주요 H2가 페이지 주제와 일치하는지
- 화면에 보이는 FAQ와 JSON-LD FAQ가 같은지
- 화면 후기 별점과 JSON-LD 후기 별점이 같은지
- 대표 이미지와 Article JSON-LD image가 맞는지
- `local`, `seoul` 이미지 alt가 `{페이지 타이틀} {홈페이지 제목}`인지
- 내부 링크/이미지 경로가 깨지지 않았는지
- `????` 같은 깨진 문자가 없는지
